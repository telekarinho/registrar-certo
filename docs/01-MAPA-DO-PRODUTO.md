## 1. Visao Geral

### 1.1 Identidade do Produto

| Campo | Descricao |
|-------|-----------|
| **Nome** | Registrar Certo |
| **Tipo** | SaaS web-first (responsivo, progressivo) |
| **Dominio** | Propriedade intelectual no Brasil via INPI |
| **Pais** | Brasil |
| **Idioma principal** | Portugues brasileiro (pt-BR) |

### 1.2 Posicionamento

> "Descubra o jeito certo de registrar sua marca, invencao ou visual de produto no INPI sem se perder."

O Registrar Certo existe para eliminar a confusao, a burocracia e a linguagem tecnica que impedem milhoes de empreendedores brasileiros de proteger aquilo que criaram. Nao somos um escritorio de advocacia. Nao somos o INPI. Somos o guia que faltava entre a sua ideia e o protocolo correto.

### 1.3 Publico-Alvo

| Persona | Perfil | Dor principal | Comportamento |
|---------|--------|---------------|---------------|
| **Empreendedor iniciante** | Dono de MEI, micro ou pequena empresa, 25-45 anos, primeiro negocio | "Quero registrar minha marca mas nao sei por onde comecar e tenho medo de errar" | Pesquisa no Google "como registrar marca", compara precos de escritorios, desiste pela complexidade |
| **Inventor independente** | Pessoa fisica com invencao ou melhoria funcional, diversas idades | "Criei algo novo mas nao sei se eh patente ou modelo de utilidade, e o processo parece impossivel" | Busca "como patentear uma ideia", confunde patente com marca, nao sabe montar documentacao tecnica |
| **Designer / Criativo** | Profissional ou estudante de design, arquitetura ou engenharia de produto | "Desenvolvi um design unico para um produto mas nao sabia que podia proteger a aparencia" | Muitos nem sabem que desenho industrial existe como categoria de protecao no INPI |
| **Startup early-stage** | Time de 2-10 pessoas, produto digital ou fisico, buscando investimento | "Investidor perguntou sobre PI e percebemos que nao protegemos nada" | Precisa resolver rapido, quer entender o basico, pode evoluir para servico premium |
| **Pequeno negocio consolidado** | Empresa com 3-10 anos, faturamento estavel, nunca registrou marca | "Uso meu nome ha anos mas nunca registrei, agora vi um concorrente com nome parecido" | Urgencia reativa, disposto a pagar por orientacao clara |

### 1.4 Problema Central

O ecossistema de propriedade intelectual no Brasil sofre de quatro barreiras criticas:

1. **Confusao conceitual**: A maioria das pessoas nao sabe a diferenca entre marca, patente e desenho industrial. Chamam tudo de "patente" ou acham que registrar marca no cartorio resolve.

2. **Burocracia intimidadora**: O sistema do INPI (e-INPI) eh funcional mas nao eh amigavel. Formularios extensos, classificacoes tecnicas (Nice, IPC, Locarno), GRUs com codigos especificos -- tudo isso afasta quem nao tem experiencia.

3. **Linguagem inacessivel**: Termos como "reivindicacoes", "relatorio descritivo", "exame formal", "sobrestamento", "oposicao" sao linguagem corrente para advogados de PI mas completamente alienantes para o publico geral.

4. **Custo percebido alto**: Muitos acreditam que precisam contratar um advogado ou agente de PI para qualquer registro, quando na verdade o INPI permite que pessoas fisicas e juridicas facam o pedido diretamente. O problema nao eh acesso -- eh orientacao.

### 1.5 Proposta de Valor

O Registrar Certo resolve essas quatro barreiras com:

- **Triagem inteligente** que identifica o tipo correto de protecao antes de qualquer formulario
- **Passo a passo humanizado** que traduz cada etapa do INPI em linguagem simples
- **Checklists acionaveis** que garantem que nada sera esquecido antes do protocolo
- **Camada de IA** que explica, resume e orienta em tempo real
- **Integracao com dados publicos do INPI** para verificacoes previas de viabilidade

## 2. Regra de Ouro

### NUNCA CONFUNDIR OS TRES TIPOS DE PROTECAO

Esta eh a regra mais importante de todo o produto. Cada tela, cada texto, cada interacao deve reforcar a distincao entre os tres tipos de protecao oferecidos pelo INPI. A confusao entre eles eh a raiz de 80% dos erros que o publico comete.

### 2.1 MARCA

| Aspecto | Descricao |
|---------|-----------|
| **O que eh** | Nome, logotipo, simbolo, sinal distintivo que identifica produtos ou servicos |
| **O que protege** | A identidade visual e nominativa do seu negocio no mercado |
| **Exemplos** | "Registrar Certo" (nome), o logotipo da empresa, um slogan, uma marca sonora |
| **Orgao no INPI** | Diretoria de Marcas |
| **Classificacao** | Classificacao de Nice (45 classes de produtos/servicos) |
| **Validade** | 10 anos, renovavel indefinidamente |
| **Custo GRU base** | R$ 166 (pessoa fisica/MEI com desconto) a R$ 415 (pessoa juridica) -- valores sujeitos a atualizacao |
| **Tempo medio** | 12 a 18 meses para deferimento (pode variar) |
| **Erro comum** | "Quero patentear minha marca" -- marca nao se patenteia, se registra |

### 2.2 PATENTE

| Aspecto | Descricao |
|---------|-----------|
| **O que eh** | Protecao para invencao (PI) ou modelo de utilidade (MU) |
| **O que protege** | Uma solucao tecnica nova para um problema tecnico -- o funcionamento, o mecanismo, o processo |
| **Exemplos** | Um novo mecanismo de fechadura (PI), uma melhoria ergonomica em ferramenta existente (MU) |
| **Orgao no INPI** | Diretoria de Patentes, Programas de Computador e Topografias de Circuitos Integrados |
| **Classificacao** | Classificacao Internacional de Patentes (IPC) |
| **Validade** | PI: 20 anos / MU: 15 anos (nao renovavel) |
| **Custo GRU base** | Varia conforme etapa (deposito, exame, expedicao) -- consultar tabela INPI vigente |
| **Tempo medio** | 3 a 8 anos para concessao (backlog historico do INPI, em reducao) |
| **Erro comum** | "Quero patentear minha ideia" -- ideias nao sao patenteaveis, apenas solucoes tecnicas concretas |

**Subtipos de patente:**

| Subtipo | Patente de Invencao (PI) | Modelo de Utilidade (MU) |
|---------|--------------------------|--------------------------|
| Requisito | Novidade + atividade inventiva + aplicacao industrial | Novidade + ato inventivo + aplicacao industrial |
| Nivel de inovacao | Alto -- algo que nao existia | Moderado -- melhoria funcional em algo existente |
| Exemplo | Novo tipo de motor eletrico | Cabo de ferramenta com encaixe aprimorado |
| Validade | 20 anos | 15 anos |

### 2.3 DESENHO INDUSTRIAL

| Aspecto | Descricao |
|---------|-----------|
| **O que eh** | Protecao para a aparencia visual ornamental de um objeto ou conjunto de objetos |
| **O que protege** | A forma, o padrao, as linhas, as cores -- aquilo que se ve, nao o que funciona |
| **Exemplos** | Design de uma garrafa, formato de um movel, padrao estampado de um tecido, interface grafica (GUI) |
| **Orgao no INPI** | Diretoria de Patentes (mesma diretoria, processo diferente) |
| **Classificacao** | Classificacao de Locarno |
| **Validade** | 10 anos, renovavel por 3 periodos de 5 anos (maximo 25 anos) |
| **Custo GRU base** | Consultar tabela INPI vigente |
| **Tempo medio** | Registro mais rapido (nao passa por exame tecnico obrigatorio, apenas formal) |
| **Erro comum** | "Quero patentear o design do meu produto" -- design se registra como desenho industrial, nao como patente |

### 2.4 Tabela Comparativa Rapida

| Criterio | Marca | Patente | Desenho Industrial |
|----------|-------|---------|--------------------|
| Protege o que? | Nome / logo / sinal | Invencao / mecanismo | Aparencia / forma visual |
| Palavra-chave | Identidade | Funcionalidade | Estetica |
| Renovavel? | Sim, indefinidamente | Nao | Sim, ate 25 anos total |
| Exame tecnico? | Sim | Sim (rigoroso) | Apenas formal |
| Classificacao | Nice | IPC | Locarno |
| Complexidade documental | Media | Alta | Media-baixa |
| Pode fazer sozinho? | Sim, com orientacao | Possivel, mas complexo | Sim, com orientacao |

### 2.5 Regras de Aplicacao no Produto

1. **Toda tela que mencione um tipo de protecao deve exibir o icone e a cor correspondente** -- nunca misturar.
2. **A triagem inteligente eh obrigatoria antes de qualquer fluxo** -- o usuario nunca deve "escolher" marca/patente/DI sem antes responder as perguntas de triagem.
3. **Alertas de confusao devem ser proativos** -- se o usuario descreve algo que soa como patente mas esta no fluxo de marca, o sistema deve alertar.
4. **Glossario contextual em toda tela** -- termos tecnicos do INPI devem ter tooltip ou explicacao inline, sempre.

## 4. Nucleos do Produto

### 4.1 Nucleo 1 -- Core (funcionalidades essenciais)

Este nucleo eh o produto minimo que entrega valor. Sem ele, nao existe Registrar Certo.

#### 4.1.1 Triagem Inteligente

**Objetivo**: Identificar com seguranca qual tipo de protecao o usuario precisa antes de qualquer formulario.

**Funcionamento**:
- Sequencia de 5 a 8 perguntas simples em linguagem coloquial
- Cada resposta refina a classificacao (marca / patente PI / patente MU / desenho industrial / nenhum / multiplos)
- Logica de arvore de decisao com pesos por resposta
- Resultado apresentado com nivel de confianca e explicacao

**Perguntas-chave da triagem**:

| # | Pergunta | Opcoes | Direciona para |
|---|----------|--------|----------------|
| 1 | O que voce quer proteger? | Nome/logo do negocio / Algo que inventei / Aparencia de um produto / Nao sei | Marca / Patente / DI / Continua triagem |
| 2 | Seu [item] ja existe no mercado de alguma forma? | Sim, mas o meu eh diferente / Nao, eh completamente novo / Nao sei | Refina entre PI e MU / Valida novidade |
| 3 | O que torna o seu [item] especial? | O jeito que funciona / O jeito que aparece visualmente / O nome ou simbolo / Mais de um | Patente / DI / Marca / Multiplos fluxos |
| 4 | Voce quer proteger a funcao tecnica ou so a aparencia? | Funcao / Aparencia / Os dois | Patente / DI / Ambos |
| 5 | Voce ja usa esse [nome/invencao/design] comercialmente? | Sim / Nao, ainda estou planejando | Afeta urgencia e orientacao |

**Resultado da triagem**:
- Tipo de protecao identificado (com icone e cor)
- Explicacao em 2-3 frases de por que esse tipo se aplica
- Alerta se mais de um tipo pode ser necessario
- Botao para iniciar o fluxo correspondente
- Opcao de salvar resultado (requer cadastro)

#### 4.1.2 Passo a Passo por Tipo

Fluxos detalhados para cada tipo de protecao (ver secao 7 para detalhamento completo).

**Principios de design dos fluxos**:
- Uma etapa por tela (mobile-first)
- Barra de progresso visivel sempre
- Botao "Por que isso eh importante?" em cada etapa
- Glossario contextual em cada termo tecnico
- Opcao de voltar sem perder progresso
- Estimativa de tempo restante

#### 4.1.3 Checklist Interativo

**Objetivo**: Garantir que o usuario tenha todos os documentos e informacoes necessarios antes de protocolar no INPI.

**Estrutura por tipo**:

- [ ] Escolher forma de apresentacao (nominativa, figurativa, mista, tridimensional)
- [ ] Identificar classe(s) de Nice aplicavel(is)
- [ ] Especificar produtos/servicos dentro da classe
- [ ] Preparar imagem do logotipo (se aplicavel) -- formato JPEG, 945x945px
- [ ] Ter CPF/CNPJ valido e regular
- [ ] Criar login no e-INPI (gov.br)
- [ ] Calcular e pagar GRU de deposito
- [ ] Preencher formulario de pedido no e-INPI
- [ ] Guardar numero do protocolo
- [ ] Acompanhar publicacao na RPI (Revista da Propriedade Industrial)

**Checklist Patente:**
- [ ] Definir se eh invencao (PI) ou modelo de utilidade (MU)
- [ ] Realizar busca previa de anterioridade
- [ ] Redigir relatorio descritivo
- [ ] Elaborar reivindicacoes
- [ ] Preparar desenhos tecnicos (se aplicavel)
- [ ] Redigir resumo
- [ ] Ter CPF/CNPJ valido e regular
- [ ] Criar login no e-INPI (gov.br)
- [ ] Calcular e pagar GRU de deposito
- [ ] Preencher formulario de deposito
- [ ] Acompanhar publicacao e prazos

**Checklist Desenho Industrial:**

- [ ] Redigir campo descritivo (opcional mas recomendado)
- [ ] Ter CPF/CNPJ valido e regular

- [ ] Preencher formulario de deposito

### 4.3 Nucleo 3 -- Busca Auxiliar (Microservico Separado)

#### 4.3.1 Visao Geral

**Objetivo**: Oferecer ao usuario uma consulta previa de marcas ja registradas ou em processo no INPI, para avaliar viabilidade antes de protocolar.

**Arquitetura**: Microservico isolado, desacoplado do monolito principal, com possibilidade de ser desligado sem impacto no produto core.

**Por que separado?**
- Depende de fonte de dados externa (tipo ampla-api ou scraping controlado da busca publica do INPI)
- Disponibilidade nao eh garantida (fonte pode mudar ou cair)
- Performance pode variar (consultas podem ser lentas)
- Questoes legais e de termos de uso podem exigir desativacao rapida
- Nao deve ser ponto de falha para o produto principal

#### 4.3.2 Funcionalidades

| Funcionalidade | Descricao | Disponivel em |
|----------------|-----------|---------------|
| Busca por nome exato | Consulta se marca com nome identico existe | Plus / Premium |
| Busca por similaridade | Consulta marcas foneticamente similares | Premium |
| Filtro por classe de Nice | Refinar resultado por classe de produto/servico | Plus / Premium |
| Filtro por status | Filtrar por ativa, em exame, arquivada, etc. | Premium |
| Resultado simplificado | Lista com nome, classe, titular, status | Plus / Premium |
| Exportar resultado | PDF com resultado da busca | Premium |

#### 4.3.3 Restricoes e Salvaguardas

- **Disclaimer obrigatorio**: "Esta busca eh apenas orientativa. Somente a busca oficial do INPI tem validade legal. Recomendamos sempre consultar diretamente no site do INPI."
- **Rate limiting**: Maximo de X consultas por usuario por dia (configuravel por plano)
- **Cache**: Resultados cacheados por 24h para mesma consulta
- **Circuit breaker**: Se a fonte externa falhar, o microservico entra em modo degradado e exibe link para busca manual no INPI
- **Toggle admin**: Pode ser desligado instantaneamente pelo painel admin sem deploy
- **Logs completos**: Toda consulta eh logada para auditoria

## 5. Camadas de Integracao

A arquitetura de integracao do Registrar Certo segue tres camadas de confiabilidade, organizadas de forma que a falha de qualquer camada superior nao comprometa a inferior.

### 5.1 Camada A -- Oficial (alta confiabilidade)

**Principio**: Dados de fonte oficial, publica e estavel.

| Integracao | Fonte | Tipo | Confiabilidade |
|------------|-------|------|----------------|
| Classificacao de Nice | INPI / OMPI / dados abertos | Tabela estatica atualizada periodicamente | Alta |
| Classificacao IPC | INPI / OMPI / dados abertos | Tabela estatica atualizada periodicamente | Alta |
| Classificacao de Locarno | INPI / OMPI / dados abertos | Tabela estatica atualizada periodicamente | Alta |
| Tabela de retribuicoes (GRU) | Site oficial INPI | Scraping controlado ou atualizacao manual | Alta (com monitoramento) |
| Links oficiais e-INPI | Site oficial INPI | URLs monitoradas | Alta (com fallback) |
| Legislacao (LPI) | Planalto.gov.br | Link estavel | Alta |
| Busca publica de marcas INPI | busca.inpi.gov.br | Interface web publica | Media (depende de disponibilidade) |
| Busca publica de patentes INPI | busca.inpi.gov.br | Interface web publica | Media |

**Tratamento de falha Camada A**:
- Se dados abertos estiverem indisponiveis: usar ultima versao cacheada + flag "dados de [data]"
- Se link oficial estiver fora: ativar fallback Camada C (instrucao manual)
- Se tabela de GRU mudar: alerta admin para revisao manual

### 5.2 Camada B -- Nao Oficial Controlada (media confiabilidade)

**Principio**: Dados de fontes nao oficiais mas uteis, com controle total de liga/desliga e fallback.

| Integracao | Fonte | Tipo | Confiabilidade | Desligavel? |
|------------|-------|------|----------------|-------------|
| Busca auxiliar de marcas | API tipo ampla-api ou equivalente | Microservico separado | Media | Sim, instantaneamente |
| Enriquecimento de dados de marca | APIs complementares | Consulta sob demanda | Media-baixa | Sim |

**Tratamento de falha Camada B**:
- Circuit breaker com threshold configuravel (ex: 3 falhas em 5 minutos = desliga)
- Fallback automatico para Camada C
- Log de todas as falhas para analise
- Admin notificado por e-mail/Slack
- Nenhuma funcionalidade Core depende da Camada B

### 5.3 Camada C -- Fallback (alta resiliencia)

**Principio**: O produto funciona 100% mesmo se todas as integracoes externas caem. A Camada C eh puro conteudo estatico e orientacao manual.

| Recurso | Descricao | Sempre disponivel? |
|---------|-----------|-------------------|
| Links diretos para INPI | URLs para busca manual, formularios, tabelas | Sim (com verificacao) |
| Instrucao manual passo a passo | Texto explicando como fazer cada acao no site do INPI | Sim |
| PDF de checklist offline | Checklist exportavel que funciona sem internet | Sim |
| Tabelas de classificacao offline | Nice/IPC/Locarno em versao estatica | Sim |
| FAQ completo | Respostas para as duvidas mais comuns | Sim |
| Orientacao por e-mail | Fallback humano para casos que o sistema nao resolve | Sim (Premium) |

**Regra fundamental**: Nenhum usuario deve ficar travado por causa de uma integracao fora do ar. A Camada C sempre oferece um caminho alternativo.

## 8. Camada IA

### 8.1 Visao Geral

A inteligencia artificial no Registrar Certo opera em 5 niveis progressivos, do mais simples ao mais complexo. Cada nivel eh independente e pode ser ativado/desativado sem afetar os demais.

**Principios da IA no produto**:
1. **A IA orienta, nao decide**: Toda sugestao da IA eh apresentada como orientacao, nunca como decisao juridica
2. **Transparencia**: O usuario sempre sabe quando esta interagindo com IA
3. **Disclaimer permanente**: "Esta orientacao eh gerada por inteligencia artificial e nao substitui consultoria juridica profissional"
4. **Humano no loop**: Para funcionalidades Premium, revisao humana complementa a IA
5. **Auditavel**: Todas as respostas da IA sao logadas para revisao e melhoria

### 8.2 Nivel 1 -- Triagem por IA

**O que faz**: Analisa as respostas do usuario na triagem e identifica o tipo de protecao com nivel de confianca.

**Implementacao**:
- Modelo: LLM (GPT-4o, Claude ou equivalente) via API
- Input: respostas da triagem + descricao em texto livre
- Output: classificacao (marca/PI/MU/DI/multiplos/inconclusivo) + confianca (%) + explicacao
- Fallback: se API indisponivel, usa arvore de decisao deterministica (regras hard-coded)

**Exemplo de output**:
> "Com base nas suas respostas, identificamos que voce provavelmente precisa de um **registro de marca** (confianca: 85%). Voce descreveu um nome que identifica seus servicos no mercado -- isso eh exatamente o que uma marca protege. Nao identificamos caracteristicas de invencao ou design visual neste caso."

### 8.3 Nivel 2 -- Explicador Contextual

**O que faz**: Fornece explicacoes em linguagem simples para qualquer termo, conceito ou etapa que o usuario nao entenda.

**Implementacao**:
- Acionado por: botao "Explicar melhor" em qualquer etapa, clique em termo do glossario, pergunta no chat
- Modelo: LLM com contexto fixo (system prompt com glossario INPI + legislacao + tom de voz RC)
- Guardrails: respostas limitadas ao escopo de PI/INPI, recusa educada para perguntas fora de escopo
- Cache: respostas para perguntas frequentes sao cacheadas para reduzir custo e latencia

**Exemplo de interacao**:
> Usuario: "O que sao reivindicacoes?"
>
> IA: "Reivindicacoes sao a parte mais importante de um pedido de patente. Pense nelas como a 'cerca' em torno da sua invencao -- elas definem exatamente o que voce esta protegendo. Quanto mais clara e especifica a cerca, melhor a protecao. Existem reivindicacoes independentes (a protecao principal) e dependentes (detalhes adicionais). No Registrar Certo, vamos te guiar para escrever cada uma na etapa 7 do fluxo de patente."

### 8.4 Nivel 3 -- Checklist Inteligente

**O que faz**: Analisa o progresso do usuario no checklist e sugere proativamente o que priorizar, alerta sobre dependencias e identifica itens que parecem incompletos ou inconsistentes.

**Implementacao**:
- Analisa contexto do usuario (tipo de protecao, classe, documentos enviados)
- Sugere ordem otimizada de tarefas
- Alerta sobre inconsistencias (ex: "Voce selecionou marca figurativa mas nao fez upload de imagem")
- Estima tempo restante com base no perfil de usuarios similares

**Exemplo**:
> "Voce ja completou 7 de 12 itens. Recomendo priorizar a busca previa (item 7) antes de pagar a GRU (item 10) -- se a busca mostrar conflito, voce pode economizar a taxa. Tempo estimado para concluir: 45 minutos."

### 8.5 Nivel 4 -- Resumo Final Pre-Protocolo

**O que faz**: Gera um resumo consolidado de tudo que o usuario preparou, destaca pontos de atencao e confirma que esta pronto para protocolar.

**Implementacao**:
- Acionado quando usuario conclui checklist (100%)
- Compila todas as informacoes fornecidas ao longo do fluxo
- Gera documento resumo em linguagem simples
- Destaca riscos identificados (nome muito generico, classe potencialmente errada, etc.)
- Oferece geracao de PDF (Plus/Premium)

**Exemplo de resumo**:
> **Resumo Pre-Protocolo -- Marca**
>
> **Marca**: "Registrar Certo"
> **Apresentacao**: Mista (nome + logotipo)
> **Natureza**: Servico
> **Classe Nice**: 42 (servicos tecnologicos e cientificos)
> **Produtos/servicos**: Plataforma SaaS de orientacao para registro de propriedade intelectual
>
> **Busca previa**: Realizada em 03/04/2026 -- nenhum conflito direto encontrado na classe 42.
>
> **Pontos de atencao**:
> - O termo "Registrar" eh descritivo/generico. A protecao sera mais forte pelo conjunto "Registrar Certo" + logotipo do que pelo nome isolado. Considere registrar como marca mista.
>
> **Proximo passo**: Acesse o e-INPI, pague a GRU (codigo XXXX, valor R$ XXX) e preencha o formulario de deposito.

### 8.6 Nivel 5 -- Motor de Conversao

**O que faz**: Analisa o comportamento do usuario no funil e gera intervencoes personalizadas para reduzir abandono e aumentar conversao.

**Implementacao**:
- Analisa padroes de uso: tempo por etapa, pontos de pausa, revisitas, abandono
- Gera mensagens contextuais: tooltips, banners, e-mails
- Sugere upgrade no momento de maior propensao (ex: quando usuario tenta gerar PDF no plano gratis)
- A/B testing de mensagens para otimizacao continua

**Tipos de intervencao**:

| Gatilho | Intervencao | Canal |
|---------|-------------|-------|
| Usuario para na etapa 5 por >5min | Tooltip: "Esta eh a etapa que mais gera duvida. Quer que eu explique de outro jeito?" | In-app |
| Usuario completa triagem mas nao cadastra | Pop-up: "Salve seu resultado para continuar depois. Leva 30 segundos." | In-app |
| Usuario cadastrado nao volta em 3 dias | E-mail: "Voce descobriu que precisa de [tipo]. Quer continuar de onde parou?" | E-mail |
| Usuario gratis tenta funcao Plus | Modal: "Esta funcao faz parte do plano Plus. Teste gratis por 7 dias." | In-app |
| Usuario Plus completa fluxo | E-mail: "Parabens! Quer que um profissional revise seus documentos antes de protocolar? Conheca o Premium." | E-mail |

### 10.2 Fase 1 -- MVP (12-16 semanas)

**Objetivo**: Validar que pessoas conseguem usar o Registrar Certo para entender o que precisam e seguir o passo a passo de registro de marca ate o protocolo no INPI.

**Hipotese central**: "Se oferecermos uma triagem inteligente + passo a passo humanizado para registro de marca, empreendedores que hoje desistem pela complexidade conseguirao completar o processo sozinhos."

**Metrica de sucesso da fase**: >= 100 usuarios completam a triagem E >= 30 iniciam o fluxo de marca E >= 10 relatam ter protocolado no INPI.

#### Entregas da Fase 1

| Entrega | Escopo | Prioridade | Estimativa | Dono |
|---------|--------|------------|------------|------|
| **Landing page** | Pagina principal com proposta de valor, CTA para triagem, SEO basico | P0 | 1 semana | Frontend |
| **Triagem inteligente v1** | 5 perguntas, arvore de decisao deterministica (sem IA), resultado com tipo + explicacao | P0 | 2 semanas | Full-stack |
| **Fluxo Marca completo** | 13 etapas com conteudo, progresso, glossario contextual | P0 | 4 semanas | Full-stack + Conteudo |
| **Checklist Marca basico** | Itens principais com toggle de marcar/desmarcar, salvo no backend | P0 | 1 semana | Full-stack |
| **Cadastro e autenticacao** | E-mail + senha, verificacao de e-mail, login, recuperacao de senha | P0 | 2 semanas | Backend |
| **Painel do usuario basico** | Ultima triagem, fluxo ativo, checklist em andamento | P0 | 1 semana | Frontend |
| **Paginas de conteudo SEO** | 5-8 paginas educativas (marca, patente, DI, diferenca, glossario) | P0 | 2 semanas | Conteudo + Frontend |
| **Links oficiais INPI** | Curadoria e publicacao de links essenciais com verificacao manual | P1 | 3 dias | Conteudo |
| **Analytics basico** | Eventos de funil core (triagem, cadastro, fluxo, checklist) | P1 | 1 semana | Frontend + Backend |
| **Infraestrutura** | Deploy, CI/CD, banco de dados, dominio, SSL, monitoramento basico | P0 | 2 semanas | DevOps/Backend |
| **LGPD basico** | Termos de uso, politica de privacidade, consentimento de cookies | P0 | 1 semana | Juridico + Frontend |

**O que NAO esta na Fase 1** (e por que):
- Fluxo de patente e DI: marca eh o caso de uso mais comum e valida o modelo
- IA: a arvore de decisao deterministica eh suficiente para validar a triagem
- Busca auxiliar: complexidade de integracao alta, valor pode ser entregue via link para INPI
- PDF: funcionalidade de retencao, nao de aquisicao
- Notificacoes por e-mail: exceto boas-vindas e verificacao
- Painel admin: gerenciamento manual via banco de dados eh aceitavel no MVP
- Monetizacao: nao cobrar ate validar que o produto funciona

## Apendice A -- Glossario de Termos INPI

| Termo | Significado em linguagem simples |
|-------|----------------------------------|
| INPI | Instituto Nacional da Propriedade Industrial -- orgao federal responsavel por marcas, patentes e desenhos industriais no Brasil |
| e-INPI | Sistema online do INPI onde voce faz todo o processo de pedido |
| GRU | Guia de Recolhimento da Uniao -- eh o "boleto" que voce paga para o governo processar seu pedido |
| RPI | Revista da Propriedade Industrial -- publicacao semanal do INPI onde saem todos os pedidos e decisoes |
| Classificacao de Nice | Sistema internacional que divide produtos e servicos em 45 classes -- voce escolhe a classe que se encaixa no seu negocio |
| IPC | Classificacao Internacional de Patentes -- sistema para categorizar invencoes por area tecnica |
| Locarno | Classificacao Internacional de Desenhos Industriais -- sistema para categorizar produtos por tipo de objeto |
| Deposito | Ato de protocolar seu pedido no INPI -- eh o "primeiro passo oficial" |
| Deferimento | Quando o INPI aprova seu pedido -- significa que voce conseguiu o registro |
| Indeferimento | Quando o INPI recusa seu pedido -- voce pode recorrer |
| Exigencia | Quando o INPI pede que voce corrija ou complemente algo no seu pedido |
| Oposicao | Quando alguem contesta seu pedido de marca -- voce tem prazo para se defender |
| Sobrestamento | Quando o INPI pausa a analise do seu pedido por causa de outro processo relacionado |
| Reivindicacoes | A parte da patente que define exatamente o que voce esta protegendo |
| Relatorio descritivo | Documento detalhado que explica sua invencao -- como funciona, como fazer, como usar |
| Anuidade | Taxa anual que voce paga para manter sua patente ativa |
| Quinquenio | Taxa paga a cada 5 anos para manter o registro de desenho industrial |
| LPI | Lei da Propriedade Industrial (Lei 9.279/1996) -- a lei que rege tudo isso |
